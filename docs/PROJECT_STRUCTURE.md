# Project Structure

`Duck` is a Rojo-managed Roblox project. The source files in this repository map into Roblox services through `default.project.json`.

## Root Files

- `default.project.json`: Rojo project tree and service mappings.
- `aftman.toml`: Aftman-managed tool versions.
- `wally.toml`: Wally package dependencies.
- `wally.lock`: Locked Wally dependency versions after package install.
- `README.md`: Project overview and quickstart.
- `AGENTS.md`: Instructions for AI coding agents.
- `.gitignore`: Generated place files, Studio locks, sourcemaps, Wally packages, and temporary generated design outputs.

## Asset Folders

### `assets/design/references`

Use for style references, sketches, screenshots, and inspiration.

### `assets/design/generated`

Use for temporary Nano Banana Pro outputs and experiments. This folder is ignored by Git because generated images can become large quickly.

### `assets/design/approved`

Use for generated or edited images approved as useful project direction.

### `assets/ui/icons`

Use for final or likely-final UI icon assets.

### `assets/ui/backgrounds`

Use for UI backgrounds that may be used in Roblox.

### `assets/ui/mockups`

Use for approved UI layout mockups.

See `docs/ASSET_WORKFLOW.md` for asset rules.

## Source Folders

### `Packages`

Mapped to:

```text
ReplicatedStorage
  Packages
```

This folder is generated by Wally and should not be edited by hand. Run `wally install` to regenerate it from `wally.toml` and `wally.lock`.

Current approved packages:

- `React`
- `ReactRoblox`

### `src/client`

Mapped to:

```text
StarterPlayer
  StarterPlayerScripts
    Client
```

Use this folder for LocalScripts and client-only modules. Client code should handle presentation, input, camera behavior, UI, and local feedback. It must not be trusted for authoritative gameplay decisions.

Current files:

- `init.client.luau`
- `App.luau`
- `Screens/FarmScreen.luau`
- `Components/ActionButton.luau`
- `Components/ResourceCounter.luau`
- `UI/Theme.luau`

### `src/server`

Mapped to:

```text
ServerScriptService
  Server
```

Use this folder for server scripts and server-only modules. Server code should own authoritative state, validation, persistence, and any logic that affects other players.

Current files:

- `init.server.luau`
- `GameServer.luau`
- `PlayerStateService.luau`

### `src/shared`

Mapped to:

```text
ReplicatedStorage
  Shared
```

Use this folder for modules that are safe to require from both client and server. Shared modules should avoid runtime assumptions that only work on one side.

Current files:

- `AssetIds.luau`
- `PrototypeConfig.luau`
- `RemoteProtocol.luau`
- `Types.luau`

## Runtime Remotes

The server creates these runtime instances under `ReplicatedStorage`:

```text
ReplicatedStorage
  DuckRemotes
    DuckAction
    DuckState
```

- `DuckAction`: client-to-server `RemoteEvent` for named UI actions. The `care_for_ducks` action may include a validated `duckIndex` payload for the selected duck.
- `DuckState`: server-to-client `RemoteEvent` for serialized farm state.

`DuckAction` includes gameplay actions and Studio-only tester actions. Tester actions must stay guarded on the server with `RunService:IsStudio()` and must not become production rewards.

The names live in `src/shared/RemoteProtocol.luau`. Do not add new remotes or payload shapes without updating the protocol, security expectations, and docs.

## Roblox Services in the Rojo Tree

The current Rojo tree defines:

- `ReplicatedStorage.Packages`

Do not change these mappings unless the change is intentional and documented.
